Как включить синхронизацию задач

Для пользователей сайта всё просто:
1. Открыть сайт.
2. Нажать "Синхронизация".
3. Ввести email и пароль.
4. Нажать "Регистрация" при первом входе или "Войти", если аккаунт уже есть.
5. На другом устройстве войти тем же email и паролем.

Дополнительно можно нажать "Войти через Google", если владелец сайта включил Google provider в Supabase.

Техническая настройка для владельца сайта делается один раз:

1. Создайте бесплатный проект на https://supabase.com.
2. Откройте в Supabase SQL Editor.
3. Выполните файл SUPABASE_SYNC_SETUP.sql.
4. Откройте Project Settings -> API.
5. Скопируйте Project URL и anon public key.
6. В файле sync-config.js вставьте эти значения:

window.MSTODO_SYNC = {
  supabaseUrl: "https://ВАШ-ПРОЕКТ.supabase.co",
  supabaseAnonKey: "ВАШ_ANON_PUBLIC_KEY"
};

7. Откройте Authentication -> URL Configuration.
8. В Site URL укажите адрес вашего сайта на GitHub Pages.
9. В Redirect URLs добавьте этот же адрес сайта.
10. Откройте Authentication -> Providers -> Email и включите Email provider.
11. Для личного простого сайта можно отключить Confirm email, чтобы регистрация сразу работала без письма.
12. Если нужен Google-вход: откройте Authentication -> Providers -> Google и включите Google provider.
13. В Google Cloud Console создайте OAuth Client для Web application.
14. В Authorized redirect URIs добавьте callback из Supabase Google provider. Он выглядит примерно так:
   https://ВАШ-ПРОЕКТ.supabase.co/auth/v1/callback
15. Скопируйте Google Client ID и Client Secret обратно в Supabase Google provider и сохраните.
16. Залейте обновлённые файлы сайта на GitHub Pages.

Важно:
- Обычные пользователи не видят Supabase Project URL и anon key.
- Sync ID не нужен. Синхронизация идёт по аккаунту Supabase Auth.
- Задачи и списки видит только вошедший пользователь.
- Используется таблица todo_user_sync. Старую таблицу todo_sync можно не трогать.
- Синхронизируются задачи и списки. Файлы-вложения остаются локально на устройстве.
- Если два устройства меняют задачи одновременно, победит последнее сохранение.
